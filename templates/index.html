Dynamic Memory Allocator with Linked List

Memory Blocks

Concepts of Dynamic Memory Allocation

Dynamic memory allocation allows programs to allocate and release memory at runtime based on current needs.

This makes memory usage more efficient compared to static memory allocation, where the size is fixed at compile time.

How Memory Allocation Works

  • Memory is requested and allocated in blocks during runtime.
  • If a block with sufficient memory is found, it is allocated to the program.
  • If the block is larger than needed, the remaining space is split into a new free block.
  • When memory is no longer needed, the allocated block is freed.
  • Any adjacent free blocks are merged to form a larger free block, preventing fragmentation.

How Linked Lists Manage Memory Blocks

A linked list is a sequence of nodes connected via pointers, used to store the memory blocks in our dynamic memory allocator:

  • Each node (block) stores:
    • Size: The amount of memory in the block.
    • Status: Whether the block is free or allocated.
    • Pointer (Next): A reference to the next block in the list.
  • The linked list allows us to efficiently traverse the memory blocks and find the best-fit block for allocation.
  • During allocation, a block may be split, and during freeing, adjacent free blocks are merged.
